# API Scope

## API Goal

The API serves precomputed model prediction output by `user_id`. It gives another engineer or reviewer a simple way to retrieve a client's term deposit subscription propensity prediction.

## Lookup API vs Realtime Inference

The MVP API is lookup-based. It does not run model inference at request time.

The PySpark pipeline is responsible for generating the prediction output table. The API reads from that output and returns the matching row by `user_id`.

## Endpoint Scope

MVP endpoint:

```http
GET /prediction/{user_id}
```

The endpoint returns one prediction response for the requested `user_id` if that ID exists in the prediction output table.

## Data Source

The API reads from the prediction output generated by the PySpark pipeline.

Required prediction output fields:

- `user_id`
- `prediction_score`
- `prediction_label`
- `model_version`
- `scored_at`
- `source`

For API and performance testing, the prediction output may be expanded or mocked to approximately 500k rows using the same schema. Expanded or mock rows must be marked with the correct `source` value and must not be claimed as real raw training users.

## Cache Scope

The MVP cache starts with an in-memory cache.

The cache should store repeated `user_id` lookups:

- Cache hit: the API finds the requested `user_id` in memory and returns the cached response faster.
- Cache miss: the API does not find the requested `user_id` in memory, reads from the prediction data source, then stores the result if appropriate.

Redis may be considered later as a production-like option, but it is not required for the MVP.

## Test Scope

Functional tests should verify:

- Successful lookup for an existing `user_id`.
- 404 response for a missing `user_id`.
- Error response shape for server failures.
- Response fields and data types.
- Cache hit and cache miss behavior if observable.

Load tests should simulate expected or normal traffic.

Stress tests should increase traffic until latency, throughput, or error rate degrades.

Metrics to collect:

- Latency.
- Throughput/RPS.
- Error rate.
- P50/P95/P99 latency.
- Cache hit rate if available.

## Out of Scope

- Realtime model inference in the API.
- Training or scoring inside the API service.
- Redis or distributed cache for MVP.
- Full production auth implementation.
- Serving raw sensitive data.
- Treating mock or expanded rows as real raw users.

## Definition of Done

- API purpose and endpoint are documented.
- Lookup-based serving behavior is clear.
- Prediction output schema is defined.
- Cache expectations are documented.
- Functional, load, and stress test scopes are documented.
- PII and privacy assumptions are documented.
- The API scope is ready for implementation planning and mentor review.
